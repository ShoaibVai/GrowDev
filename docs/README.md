## ğŸ“ Documentation Standards

When adding new documentation:

1. **Use Markdown** - All docs use `.md` format
2. **Include ToC** - Add table of contents for long docs
3. **Use Emojis** - Make docs visually scannable
4. **Code Examples** - Include practical examples
5. **Keep Updated** - Update date at bottom
6. **Link Related Docs** - Cross-reference related content
7. **Test Commands** - Verify all commands work

### Template Structure

```markdown
# Document Title

Brief introduction paragraph.

## Table of Contents
- [Section 1](#section-1)
- [Section 2](#section-2)

## Section 1
Content...

## Section 2
Content...

---

Last updated: YYYY-MM-DD
```
